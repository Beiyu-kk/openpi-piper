# Piper pi0/pi05 训练启动全流程

本文档记录当前 openpi 项目中 Piper 右臂抓书任务的训练流程。兼容旧 checkpoint 的默认训练配置名为：

```text
pi05_piper_right_book_v5_lora
```

夹爪也使用 delta action 的新实验配置名为：

```text
pi05_piper_right_book_v5_lora_all_delta
```

数据集路径在代码配置中已经写入：

```text
/mnt/c9dd2903-1a5c-4ec3-b146-9f8ee2434744/Dataset/piper_data/data/lerobot_v21/piper_right_book_noRGBD
```

该数据集是 LeRobot v2.1 格式，包含主视角 `top_head`、右腕相机 `hand_right`、7 维状态、7 维动作，以及保存在 task 中的中文 prompt。

## 1. 进入项目环境

打开终端，进入 openpi 仓库根目录：

```bash
cd <openpi_repo>
```

其中 `<openpi_repo>` 是包含 `pyproject.toml`、`scripts/`、`src/`、`assets/`、`checkpoints/` 的目录。

训练使用 openpi 仓库自带的 `uv` 环境，不使用 `piper-openpi` conda 环境。`piper-openpi` 只用于机械臂推理控制端。

## 2. 默认训练配置

配置位置：

```text
src/openpi/training/config.py
```

默认 Piper 训练配置为：

```python
TrainConfig(
    name="pi05_piper_right_book_v5_lora",
    model=Pi0Config(
        pi05=True,
        action_dim=32,
        action_horizon=15,
        paligemma_variant="gemma_2b_lora",
        action_expert_variant="gemma_300m_lora",
    ),
    weight_loader=CheckpointWeightLoader("gs://openpi-assets/checkpoints/pi05_base/params"),
    ema_decay=None,
    num_train_steps=30_000,
    save_interval=5_000,
    keep_period=5_000,
    batch_size=32,
    num_workers=4,
)
```

说明：

- 基座模型：`pi05_base`
- 微调方式：LoRA
- LoRA 位置：PaliGemma 和 action expert 两侧都启用 LoRA
- 冻结策略：冻结非 LoRA 主干权重，只训练 LoRA 参数
- 动作块长度：`action_horizon=15`
- 训练步数：`30000`
- checkpoint 保存间隔：每 `5000` step 保存一次
- 默认 batch size：代码里是 `32`
- 实机之前验证过 `batch_size=96` 可启动；如果要更吃满显存，推荐用命令行覆盖到 `96`

## 3. 数据与动作处理

Piper 数据 transform 位置：

```text
src/openpi/policies/piper_policy.py
```

字段映射：

```text
observation.images.top_head   -> base_0_rgb
observation.images.hand_right -> right_wrist_0_rgb
left_wrist_0_rgb              -> 零图占位，mask=False
observation.state             -> state
action                        -> actions
prompt                        -> prompt
```

当前动作处理：

```python
DeltaActions(make_bool_mask(6, -1))
AbsoluteActions(make_bool_mask(6, -1))
```

含义：

- 旧配置 `pi05_piper_right_book_v5_lora`：前 6 维关节训练时从绝对位置转成相对动作，推理输出后再转回绝对位置
- 第 7 维夹爪保持连续绝对值，不做 delta
- 新配置 `pi05_piper_right_book_v5_lora_all_delta`：7 维动作全部做 delta，推理输出后全部转回绝对位置
- 夹爪输出：不做 `0/1` 阈值化，保持连续输出

注意：如果重新训练时继续使用 delta 动作，建议使用 delta 后的相对 `actions` norm stats。不要把绝对动作统计和 delta 动作训练混用。

## 4. 检查 norm stats

当前 norm stats 默认读取位置：

```text
assets/pi05_piper_right_book_v5_lora/piper_right_book_v5/norm_stats.json
```

检查文件是否存在：

```bash
test -f assets/pi05_piper_right_book_v5_lora/piper_right_book_v5/norm_stats.json && echo "norm stats ok"
```

查看前 7 维统计：

```bash
python - <<'PY'
import json
from pathlib import Path

p = Path("assets/pi05_piper_right_book_v5_lora/piper_right_book_v5/norm_stats.json")
data = json.loads(p.read_text())["norm_stats"]

for key in ["state", "actions"]:
    print(f"\n{key}")
    for stat in ["mean", "std", "q01", "q99"]:
        print(stat, data[key][stat][:7])
PY
```

如果 `actions` 前 7 维统计和 `state` 非常接近，说明它还是绝对动作统计；如果继续使用 delta 动作训练，建议按下一节重算。

## 5. 可选：重算相对动作 norm stats

如果要使用“delta 动作训练 + 相对 norm stats”，先备份当前统计：

```bash
cp assets/pi05_piper_right_book_v5_lora/piper_right_book_v5/norm_stats.json \
   assets/pi05_piper_right_book_v5_lora/piper_right_book_v5/norm_stats.absolute_backup.json
```

然后重新计算：

```bash
uv run --no-dev scripts/compute_norm_stats.py \
  --config-name=pi05_piper_right_book_v5_lora
```

输出仍会写到：

```text
assets/pi05_piper_right_book_v5_lora/piper_right_book_v5/norm_stats.json
```

重算后再次查看前 7 维统计。期望结果：

- `state` 仍是绝对关节位置统计
- 旧配置：`actions` 前 6 维变成相对动作统计，第 7 维夹爪保持连续绝对值统计
- 新配置：`actions` 7 维全部变成相对动作统计，第 7 维夹爪统计也变成相对夹爪位移

重要：更换 norm stats 后，不建议直接拿旧 checkpoint 继续对比推理效果。推荐从 `pi05_base` 重新开始一个新的实验名训练。

## 6. 推荐启动训练命令

推荐先使用 `batch_size=96`，限制 JAX 使用 95% 显存：

```bash
cd <openpi_repo>

XLA_PYTHON_CLIENT_MEM_FRACTION=0.95 \
uv run --no-dev scripts/train.py pi05_piper_right_book_v5_lora \
  --exp-name=piper_right_book_v5_lora_bs96 \
  --batch-size=96 \
  --overwrite
```

说明：

- `XLA_PYTHON_CLIENT_MEM_FRACTION=0.95`：限制 JAX 最多预分配约 95% GPU 显存，避免系统卡死
- `--exp-name`：本次训练输出目录名
- `--batch-size=96`：覆盖 config 里的默认 `32`
- `--overwrite`：如果同名实验目录已存在，会删除旧目录重新训练
- 不要同时使用 `--overwrite` 和 `--resume`

输出目录：

```text
/mnt/c9dd2903-1a5c-4ec3-b146-9f8ee2434744/checkpoints/openpi/pi05_piper_right_book_v5_lora_joint_delta_gripper_absolute/piper_right_book_joint_delta_gripper_absolute_bs32
```

## 8. OOM 调参方式

如果 `batch_size=96` OOM，优先降低 batch size：

```bash
XLA_PYTHON_CLIENT_MEM_FRACTION=0.95 \
uv run --no-dev scripts/train.py pi05_piper_right_book_v5_lora \
  --exp-name=piper_right_book_v5_lora_bs64 \
  --batch-size=64 \
  --overwrite
```

如果 `64` 仍 OOM，再降到 `32`：

```bash
XLA_PYTHON_CLIENT_MEM_FRACTION=0.95 \
uv run --no-dev scripts/train.py pi05_piper_right_book_v5_lora \
  --exp-name=piper_right_book_v5_lora_bs32 \
  --batch-size=32 \
  --overwrite
```

不要为了省显存改小 `action_horizon`，因为当前模型和部署链路按 `action_horizon=15` 对齐。

## 9. 恢复训练

如果训练中断，且实验目录还在，可以从最近 checkpoint 恢复：

```bash
cd <openpi_repo>

XLA_PYTHON_CLIENT_MEM_FRACTION=0.95 \
uv run --no-dev scripts/train.py pi05_piper_right_book_v5_lora \
  --exp-name=piper_right_book_v5_lora_bs96 \
  --batch-size=96 \
  --resume
```

恢复训练时不要加 `--overwrite`。

## 12. 训练完成后启动推理

训练完成后，选择一个 checkpoint step，例如：


如果最终目录不是 `29999`，就把下面命令中的 step 目录替换为实际存在的目录。

```bash
cd <openpi_repo>

XLA_PYTHON_CLIENT_MEM_FRACTION=0.95 \
uv run --no-dev scripts/serve_policy.py \
  --port=8000 \
  policy:checkpoint \
  --policy.config=pi05_piper_right_book_v5_lora \
  --policy.dir=/mnt/c9dd2903-1a5c-4ec3-b146-9f8ee2434744/checkpoints/openpi/pi05_piper_right_book_v5_lora_joint_delta_gripper_absolute/piper_right_book_joint_delta_gripper_absolute_bs32/29999
```

## 13. 常见问题

### 13.1 找不到 norm stats

报错类似：

```text
Normalization stats not found.
```

处理方式：

```bash
uv run --no-dev scripts/compute_norm_stats.py \
  --config-name=pi05_piper_right_book_v5_lora
```

### 13.2 torchcodec 或 FFmpeg 报错

当前 Piper 数据配置已经设置 `video_backend="pyav"`，正常不会走 torchcodec。如果仍然报视频解码错误，先确认 `src/openpi/training/config.py` 中 `LeRobotPiperDataConfig.create()` 返回的 `DataConfig` 包含：

```python
video_backend="pyav"
```

### 13.3 wandb 网络问题

可以离线训练：

```bash
WANDB_MODE=offline XLA_PYTHON_CLIENT_MEM_FRACTION=0.95 \
uv run --no-dev scripts/train.py pi05_piper_right_book_v5_lora \
  --exp-name=piper_right_book_v5_lora_bs96 \
  --batch-size=96 \
  --overwrite
```

### 13.4 训练很慢

先看日志中的 `rate`：

```bash
tail -f logs/pi05_piper_right_book_v5_lora_bs96.log
```

影响速度的主要因素：

- batch size 越大，单步可能越慢，但每步看到的数据更多
- 第 0 步需要 JAX 编译，耗时明显更长
- 视频解码、CPU workers、磁盘读取都会影响吞吐
- 当前是 LoRA 训练，但 pi05 仍然较大，单卡训练 30000 step 需要较长时间

### 13.5 绝对动作训练还是 delta 动作训练

当前项目配置是 delta 动作训练：

```text
旧配置：前 6 维关节 absolute -> delta -> model -> absolute，第 7 维夹爪保持连续绝对值
新配置：7 维动作全部 absolute -> delta -> model -> absolute
```

如果后续想对齐 Kai0 的较好模型效果，可以新增一个不使用 `DeltaActions` 的专用 config，直接训练绝对动作，并配套使用绝对动作 norm stats。不要在同一个实验里混用两套动作定义。
